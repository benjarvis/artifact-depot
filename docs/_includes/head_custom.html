{%- comment -%} Automatic light/dark mode based on the user's OS / browser preference (`prefers-color-scheme`). just-the-docs builds three stylesheets at site build time: /assets/css/just-the-docs-default.css - whatever `color_scheme:` points at in _config.yml (we use `light`) /assets/css/just-the-docs-light.css - explicit light palette /assets/css/just-the-docs-dark.css - explicit dark palette The default (light) stylesheet is loaded unconditionally by the parent `head.html` include. We add the dark stylesheet AFTER it with a `media="(prefers-color-scheme: dark)"` attribute. The browser downloads both but only applies dark when the media query matches; the CSS cascade puts dark's rules after default's, so dark wins on match. Pure CSS, no JS, no flash-of-light-content. There is no UI toggle -- the site follows the OS/browser exactly. {%- endcomment -%}
